# 流程化产品开发体系

## 概述

本目录包含完整的流程化产品开发体系，适用于一般性产品开发（Web应用、移动应用、桌面应用等）。

## 适用场景

- 开发Web应用、移动应用、桌面应用等一般性产品
- 需要完整的需求分析、设计、开发、测试、部署流程
- 需要严格的文档规范和质量控制机制
- 需要专业的团队协作和流程管理

## 流程概览

```
需求阶段 → 设计阶段 → 开发阶段 → 测试阶段 → 部署阶段
    ↓          ↓          ↓          ↓          ↓
 PRD文档   设计文档   产品代码   测试报告   部署方案
```

### 需求阶段

- **产品需求分析**: 通过用户研究、需求调研、竞品分析等方法了解用户需求
- **PRD编写**: 编写产品需求文档，包括产品概述、用户画像、功能需求、非功能需求
- **需求评审**: 评审PRD文档，确保需求清晰、完整、可行
- **人类确认**: 用户确认PRD文档，进入设计阶段

### 设计阶段

- **UI/UX设计**: 设计用户体验和视觉界面，包括信息架构、交互设计、视觉设计
- **技术架构设计**: 设计技术架构，包括系统架构、技术选型、API设计、数据库设计
- **设计评审**: 评审设计文档和技术方案
- **人类确认**: 用户确认设计方案，进入开发阶段

### 开发阶段

- **前端开发**: 开发前端界面，实现用户交互和视觉效果
- **后端开发**: 开发后端服务，实现业务逻辑和数据处理
- **代码审查**: 审查代码质量、架构一致性、最佳实践
- **人类确认**: 用户确认开发完成情况，进入测试阶段

### 测试阶段

- **测试计划**: 制定测试策略和测试计划
- **测试执行**: 执行测试用例，记录测试结果和缺陷
- **缺陷修复**: 修复缺陷，重新测试
- **人类确认**: 用户确认测试结果和产品质量，进入部署阶段

### 部署阶段

- **部署准备**: 准备部署环境、配置、脚本
- **部署执行**: 执行部署流程
- **部署验证**: 验证部署结果，确保产品正常运行
- **人类验收**: 用户验收产品，产品开发完成

## Agent列表

### 需求阶段

- **product-manager-agent** - 产品经理
  - 职责: 产品需求分析、产品规划、PRD编写
  - 输出: PRD文档、需求分析报告、用户研究报告、竞品分析报告

### 设计阶段

- **ui-ux-designer-agent** - UI/UX设计师
  - 职责: 用户体验设计、交互设计、视觉设计、原型设计
  - 输出: 设计文档、设计稿、原型、设计规范

- **tech-lead-agent** - 技术负责人
  - 职责: 技术架构设计、技术选型、代码审查、技术规范制定
  - 输出: 技术方案、架构文档、技术规范

### 开发阶段

- **frontend-developer-agent** - 前端开发工程师
  - 职责: 前端界面开发、组件开发、前端架构
  - 输出: 前端代码、组件库、前端文档

- **backend-developer-agent** - 后端开发工程师
  - 职责: 后端服务开发、API设计、数据库设计
  - 输出: 后端代码、API文档、数据库设计文档

### 测试阶段

- **qa-engineer-agent** - 测试工程师
  - 职责: 测试计划、测试用例编写、测试执行、质量保障
  - 输出: 测试计划、测试用例、测试报告、质量报告

### 协调者

- **ProductDevelopmentOrchestrator** - 产品开发流程协调者
  - 职责: 协调整个产品开发流程
  - 功能: 调用各个Agent、验证文档质量、管理流程进度

## 使用方法

### 1. 启动协调者

调用 `ProductDevelopmentOrchestrator`，它会引导你完成整个流程。

### 2. 提供产品需求

以文本或文档形式提供你的产品需求。

### 3. 跟随流程

协调者会按照流程依次调用各个Agent：

- 每个阶段完成后需要人类确认才能进入下一阶段
- 可以在任何阶段提出修改意见
- 支持质量门禁机制确保质量

### 4. 获得产出

最终获得：

- 完整的PRD文档和设计文档
- 可运行的产品代码
- 完整的测试报告和部署方案

## 文档规范

所有Agent必须遵循以下规范：

### 1. 使用统一的文档模板

见 `design-standard/templates.md`，包含：

- 需求阶段文档模板
- 设计阶段文档模板
- 开发阶段文档模板
- 测试阶段文档模板
- 部署阶段文档模板

### 2. 执行强制的合规自检流程

每个Agent生成文档后必须：

- 写作前绑定模板
- 落盘写入
- 自动回读校验
- 合规自检
- 向协调者报告

### 3. 遵循核心原则

见 `design-standard/product-core.md`，包含：

- Agent协作设计规范
- 信息传递与上下文管理
- 职责边界与防越俎代庖
- 产品管理原则
- 设计原则
- 技术架构原则
- 开发阶段规范（代码质量、测试质量、运维质量、性能优化）

### 4. 遵循工作流程标准

见 `design-standard/workflow-standard.md`，包含：

- 产品开发流程总览
- 各阶段工作流程
- 各阶段依赖关系

## 与Agent产品开发流程的区别

| 维度         | 流程化产品开发体系（本目录）   | Agent产品开发流程（`/docs/agent-product-development/`） |
| ------------ | ------------------------------ | ------------------------------------------------------- |
| **定位**     | 用于一般性产品开发             | 专门用于开发AI Agent产品                                |
| **适用场景** | Web应用、移动应用、桌面应用    | ReAct Agent、Function Calling Agent                     |
| **特有阶段** | 无                             | 战略设计阶段（Prompt构造块、工具设计）                  |
| **设计原则** | 流程驱动、用户体验、功能完整性 | 能力驱动、ReAct Loop、工具必要性                        |
| **测试重点** | 功能测试、性能测试、兼容性测试 | Prompt测试、工具测试、Token消耗                         |
| **优化重点** | 性能优化、用户体验优化         | Prompt优化、工具调用优化、成本优化                      |

## 常见问题

### Q: 这个流程适合开发什么类型的产品？

A: 适合开发一般性产品，如Web应用、移动应用、桌面应用等。如果你要开发AI Agent产品，请使用 `/docs/agent-product-development/` 目录下的Agent产品开发流程。

### Q: 必须完整走完所有阶段吗？

A: 建议完整走完所有阶段以确保质量。但如果你已经有明确的需求和设计，可以跳过某些阶段。每个阶段完成后都需要人类确认才能进入下一阶段。

### Q: 如何确保文档质量？

A: 每个Agent都有强制的合规自检流程，包括模板匹配、章节完整性检查、必填字段检查等。协调者会验证文档质量，不合格的文档会要求重新生成。

### Q: 这个流程和敏捷开发有什么关系？

A: 这个流程可以与敏捷开发结合使用。你可以将整个产品分解为多个迭代，每个迭代使用这个流程完成需求、设计、开发、测试、部署。

### Q: 可以自定义Agent吗？

A: 可以。你可以根据项目需要添加新的Agent或修改现有Agent的职责。只需遵循统一的Agent文档结构和规范即可。

## 贡献指南
