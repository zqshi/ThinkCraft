## 4. 组件库

### 4.1 按钮组件

**主要按钮 (Primary Button)**:
- **用途**: 主要操作，如提交、确认、保存
- **视觉**: 使用主色调，白色文字，圆角设计
- **尺寸**: 大(48px)、中(40px)、小(32px)
- **状态**: 默认、悬停、点击、禁用、加载中

**次要按钮 (Secondary Button)**:
- **用途**: 次要操作，如取消、返回
- **视觉**: 透明背景，主色调边框和文字
- **尺寸**: 与主要按钮一致
- **状态**: 默认、悬停、点击、禁用

**文字按钮 (Text Button)**:
- **用途**: 辅助操作，如查看详情、了解更多
- **视觉**: 无背景无边框，主色调文字
- **状态**: 默认、悬停、点击、禁用

### 4.2 表单组件

**输入框 (Input)**:
- **类型**: 文本、数字、密码、搜索
- **状态**: 默认、聚焦、填充、错误、禁用
- **辅助元素**: 标签、占位符、帮助文本、错误提示

**下拉选择 (Select)**:
- **样式**: 单选、多选、搜索型
- **状态**: 默认、展开、选中、禁用
- **选项**: 文字、图标+文字、分组

**复选框/单选框 (Checkbox/Radio)**:
- **样式**: 标准、卡片式、按钮式
- **状态**: 未选、选中、禁用
- **布局**: 垂直、水平、网格

### 4.3 卡片组件

**标准卡片**:
- **结构**: 图片/图标 + 标题 + 描述 + 操作按钮
- **样式**: 圆角、阴影、边框
- **交互**: 悬停效果、点击反馈

**信息卡片**:
- **用途**: 展示统计数据、关键信息
- **结构**: 图标 + 数值 + 标签 + 趋势
- **样式**: 简洁、突出数值

### 4.4 导航组件

**顶部导航**:
- **结构**: Logo + 导航项 + 用户菜单
- **样式**: 固定/滚动、透明/实色
- **响应式**: 移动端转为汉堡菜单

**侧边导航**:
- **结构**: 导航项 + 图标 + 标签
- **样式**: 展开/收起、高亮当前项
- **层级**: 支持二级/三级菜单

### 4.5 反馈组件

**提示消息 (Toast)**:
- **类型**: 成功、警告、错误、信息
- **位置**: 顶部居中、右上角
- **行为**: 自动消失、可手动关闭

**对话框 (Modal)**:
- **类型**: 确认、表单、信息展示
- **样式**: 居中、圆角、遮罩
- **操作**: 确认、取消、关闭

**加载状态 (Loading)**:
- **类型**: 全屏加载、局部加载、按钮加载
- **样式**: 旋转图标、进度条、骨架屏
- **文案**: 加载提示文字

## 6. 响应式设计

### 6.1 断点设置
- **Mobile**: < 768px (手机)
- **Tablet**: 768px - 1024px (平板)
- **Desktop**: > 1024px (桌面)
- **Large Desktop**: > 1440px (大屏)

### 6.2 适配策略

**移动端 (< 768px)**:
- 单列布局，内容垂直排列
- 导航改为汉堡菜单
- 字体大小适当缩小（14px基准）
- 触摸区域至少44x44px
- 隐藏次要信息，突出核心功能

**平板端 (768px - 1024px)**:
- 两列布局，合理利用空间
- 保留主要导航，次要导航可收起
- 字体大小适中（15px基准）
- 支持触摸和鼠标操作

**桌面端 (> 1024px)**:
- 多列布局，充分利用屏幕空间
- 完整导航，所有功能可见
- 标准字体大小（16px基准）
- 优化鼠标悬停效果

## 7. 设计原则

### 7.1 核心原则
1. **简洁明了**: 界面简洁，信息清晰，避免冗余
2. **一致性**: 保持视觉和交互的一致性
3. **反馈及时**: 操作后立即给予明确反馈
4. **容错性**: 防止错误发生，易于纠正错误
5. **效率优先**: 减少用户操作步骤，提高效率
