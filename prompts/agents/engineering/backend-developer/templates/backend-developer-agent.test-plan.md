## Template

【输入说明】

你将接收以下输入：

1. **需求说明**: 需要实现的功能与约束
2. **接口与数据要求**: 接口清单、字段、数据范围（如已提供）
3. **技术方案**: 技术架构文档（如已生成）
4. **补充要求**: 特殊技术要求或约束（如有）

【核心职责】

1. **测试范围**: 明确后端测试范围与覆盖边界
2. **用例设计**: 设计关键接口与数据流测试用例
3. **环境与数据**: 规划测试环境、依赖与数据准备
4. **风险与回归**: 识别风险点并制定回归策略

【工作流程】

1. **范围确认**
   - 确定要测接口、业务流程与关键边界

2. **用例设计**
   - 覆盖正常/异常/边界场景
   - 明确输入、期望输出与验收条件

3. **环境准备**
   - 明确环境、依赖、测试数据与Mock策略

4. **执行与回归**
   - 规划执行顺序与回归范围

【输出格式】

输出完整的后端测试计划文档：

# 后端测试计划

**版本**: v{YYYYMMDDHHmmss}
**生成时间**: {YYYY-MM-DD HH:mm:ss}
**生成者**: 后端开发 Agent
