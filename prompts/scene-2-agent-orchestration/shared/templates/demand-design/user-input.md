## 使用说明

### 何时使用？

- 需求设计阶段的第一步
- 收集用户原始需求时
- 记录需求提出者的原始表述

### 填写要点

1. **保留原始表述**: 尽可能保留用户的原始表述，不要过度解读
2. **场景具体化**: 使用场景要具体，包含触发条件、用户行为、期望结果
3. **痛点量化**: 痛点要量化影响程度和发生频率
4. **优先级合理**: 优先级要基于业务价值和紧急程度综合判断

### 后续流程

1. 填写本模板后，进入需求澄清环节
2. 使用"需求澄清问题模板"生成澄清问题列表
3. 与用户沟通后，使用"需求澄清分析文档模板"整合结果
